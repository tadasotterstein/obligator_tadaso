Git og Github

Git er laga for å distribuere versonskontroll over mange system. Det Git spessialiserer seg i er å gjære det enklare for utviklere å samarbeide. Det som også er merkverdig, er at det blir også mykje lettare å jobbe på det same prosjektet på fleire datamaskiner. Ein annan god ting med github er at den har backup på filane dine. Git kan til og med finne feil i filane du har skrive. ordet versjonskontroll har blitt nemnt i denne teksten, men kva betyr det eigentlig?

versjonskontroll

Ein kan samanlikne versjonskontroll med office 365 sin tjeneste som gir tilgang til fleire brukere for å endre på tekst samtidig. Det den faktisk gjer er at den legger til kode frå ulike dokument til 1 prosjekt. dette gir også moglegheita til at du (muligens med fleire) kan jobbe fra forskjellege plassar. dette kan tilby måtar ein kan jobbe på og gjer lagringa di lettare og meir praktisk alt i alt er versjonskontroll ein trygg, lett og meir praktisk måte ein utvikler kan lagre prosjektet sitt på. det som også er sært tiltrekkande med git og github er at programmet lagrar fleire versjonar av koden din i tilleg til fleire backups. så viss du til døme har jobba på prosjektet ditt og angra på mykje av det du har skrive nylig, så kan du berre gå tilbake til ein tidlegare versjon og få ein tidligare versjon. ein kan samanlikne dette med savepoints i dataspel.

Github

Github er er ein programmet som lagrer dokumenta dine på skyen. som sagt der er berre ein praktisk måte for utviklere å lagre prosjekta sine. Github speler i samband med Git og derfor kan verke som ein god måte å lagre prosjekta sine på.

setup og repositories

Når det gjelder setup av Git og Github kan ein berre google-søke "github downlaod" og ta den første linken som kommer opp. nedlastinga går fort og fint. når du har det instalert, trenger ein å lage ein bruker, og sette opp ein repository. repositories lagar man ved å velge mappe og det som er i mappen blir oppdaga av github og da er du stort sett ferdig. Selfølgelig er det mange avanserte elementer du kan redigere på, men når det gjelder basic setup er ein allerede godt i gong.

Mine tankar om Git og Github

Git og github høyres veldig bra ut, særlig for utviklere, men eg personlig ser ikkje så stor nytte i å bruke det akuratt nå. sidan eg er ein student har eg fått 1TB lagringsplass gratis på One Drive. det gjer eigentleg det samme for meg laster opp dokumenter på skyen,¨ som gir meg tilgang til filane på kva som helst pc så lenge eg har internet. Det som høyrest attraktivt ut er versjonskontroll. tilbudet at eg kan gå tilbake til tidlegare punkt i prosjektet mitt kan spele ei viktig rolle i webutvikling, men sidan eg har allerede lagd god struktur i OneDrive, trur eg fortsatt holder meg til OneDrive.
