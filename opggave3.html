Layouts i CSS

Flexbox

Flexbox var laga for å gi ein webutviklar ein effektiv måte å legge ut nettsida si på ein fin, oversiktleg måte. Flexbox spessialiserar seg på å distribuere plass til elementar til og med når den ikkje veit kor store elemtane er. det er derfor den blir kalla for flex. poenget med flexbox er å gi wrapperen / containeren evnen til å endre på elementane (høgde og breidde). flexbox utvid elementane for å fylle opp plassen i containeren. dette kan du sjå i indexen til denne nettsida. Flexbox elementet i midten brukar litt meir plass for at den skal fylle ut alt. den kan også minske elementane for at den skal hindre at dei går i kvarandre. hovudgrunnlaget til flexbox er å gjer elementane fleksible. og flexbox gjer jobben ganske bra. for døme kan du gå på index og justere på størrelsen til nettlesaren og da går flexbox elementane loddrett for å unnga at dei blir gøymt vekk på grunn av størrelse.

CSS Grid

CSS grid er den nyeste måten å stilere ei nettside på. det går ut på at ein lager ein container med div class="" for elementar du skal stilere med grid. denne måten gir deg veldig mykje frihet på korleis ein lager ei nettside. det grid gjer er at den "deler" opp nettsida i mange deler og legger ein element i kvar. fordelene med dette er at du får ein nettside som skalerer nesten automatisk, den gjør det automatisk ryddig og gjær det veldig lett å tilpasse, legge inn, eller å berre style det godt. denne nettsida har mykje css grid i seg, navigasjons baren min for eksempel er laga i grid. det som kanskje er det beste med grid er kor lett det er å berre leggje elementar inn og stilere det med CSS grid. til og med nybegynnere kan ta bruk av grid og gjære ein ganske grei jobb med nettsida. CSS grid er den sterkaste layout systemet ute akuratt nå på grunn av at den takler både kolonner og rader noko som Flexbox ikkje klarer. Flexbox verkar meir som ein 1-dimensjonel system, mens grid kan bli brukt til meir enn det. Når eg brukte grid la eg inn ein forelderelement som heitte wrapper, og den med child-elementane mine som kan bli kalla for grid items.

her er nokre illustrasjonsbilete som viser flexbox og grid. her ser vi at flex containeren verkar som ein forelder element til flex item. den går vassrett og tar opp så mykje plass som mogleg, nett som flexboxen i index. når det gjelder grid ser vi at alt er under ein grid. det er derfor CSS grid er kanskje det beste alternativet for layouts akuratt nå, nettopp på grunn av at du kan endre på alt under 1 grid, og viss ein vil bli meir spessifik, så kaller du på child-elementet for eksempel .main og gjer kva du vil med den.
